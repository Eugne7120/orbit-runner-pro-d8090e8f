# PHASE 3 — BUILD `/app/workers` (Worker Dashboard)

## Context

You are now building the authenticated application page:

```
/app/workers
```

This is **NOT** a landing page.

This is **NOT** a marketing page.

This is the internal dashboard used by contributors after logging into 0RBIT.

A contributor uses this page to:

* Download the Worker Runtime
* Connect their wallet
* Pair their machine
* Monitor GPU status
* Watch incoming inference jobs
* View earnings
* View completed jobs
* Verify on-chain settlements
* Monitor network health

Everything should feel like operating modern cloud infrastructure.

Think:

* GitHub Actions
* Cloudflare Dashboard
* Railway
* Vercel
* Stripe Dashboard

NOT crypto dashboards.

NOT mining dashboards.

NOT DePIN dashboards.

# Implementation Notes

* Do **not** implement a real backend yet.
* Build everything as reusable React components.
* Separate components for RuntimeFlow, WorkerStatus, LiveJobs, RuntimeTerminal, PaymentFlow, OnChainProof, EarningsChart, and ActivityFeed.
* All animations should be implemented with code (CSS, Framer Motion, SVG, Canvas where appropriate). **Do not use video assets or image assets.**
* The entire `/app/workers` page should feel polished enough that a visitor believes the runtime already exists, while remaining visually honest that the data is currently simulated for the MVP.
