# 0RBIT — UI Enhancement
## Scroll Reveal Animation + SVG Logo Integration

Do NOT redesign the website.

Do NOT change the layout.

Do NOT modify the design system.

The goal of this update is to improve the overall user experience with premium scroll animations and replace all temporary branding assets with the official 0RBIT logo.

---

# Part 1 — Scroll Reveal Animation

Implement smooth scroll reveal animations across the entire website.

Every major section should animate into view only when it enters the viewport.

Avoid animations that trigger repeatedly while scrolling.

The animation should play once and remain visible.

---

## Reveal Style

Use a subtle premium animation.

Preferred animation:

Fade In + Move Up

Animation example:

- opacity: 0 → 1
- translateY: 32px → 0
- slight blur → sharp
- duration: 700–900ms
- ease: ease-out

The animation should feel elegant and modern.

Never dramatic.

---

## Staggered Animation

When multiple cards appear together,
do not animate them simultaneously.

Reveal each card with a small delay.

Example:

Card 1

↓

80ms

↓

Card 2

↓

80ms

↓

Card 3

↓

80ms

↓

Card 4

This creates a premium cascading effect.

---

## Components That Should Reveal

Apply reveal animations to:

- Hero secondary content
- Product cards
- Feature cards
- Documentation sections
- API cards
- Worker cards
- Dashboard widgets
- Pricing cards
- FAQ items
- CTA sections
- Footer content

Do NOT animate:

Navbar

Background

Continuous runtime animations

---

## Performance

Use Intersection Observer or an equivalent performant solution.

Avoid expensive scroll listeners.

Animations should remain smooth even on lower-end devices.

Respect prefers-reduced-motion.

---
