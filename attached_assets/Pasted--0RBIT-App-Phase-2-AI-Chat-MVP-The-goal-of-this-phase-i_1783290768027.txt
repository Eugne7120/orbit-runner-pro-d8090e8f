# 0RBIT App
## Phase 2 — AI Chat MVP

The goal of this phase is to build the first working version of 0RBIT Chat.

This is the core product of the entire 0RBIT ecosystem.

The experience should feel modern, premium, fast, and production-ready.

The application is hosted separately from the marketing website at:

app.0rbit.ai

Do NOT redesign the existing app layout created in Phase 1.

Extend it by building the Chat experience.

---

# Objective

Build a fully functional AI chat interface.

Backend LLM implementation can temporarily use an external provider.

However, the frontend MUST NOT expose the provider name.

Users should only see the 0RBIT models.

This allows backend providers to be replaced in the future without changing the UI.

---

Conversation History

----------------------------

Build a conversation sidebar.

Include:

- New Chat
- Conversation History
- Search Conversations
- Rename Conversation
- Delete Conversation
- Pinned Chats (optional)

Group conversations by:

Today

Yesterday

Previous 7 Days

Older

Use dummy data if backend is not ready.

---

# Top Bar

Display:

Current Chat Title

Model Selector

New Chat Button

Responsive Menu

Keep the design minimal.

---

# Model Selector

Never expose backend providers.

Only show 0RBIT branding.

Models:

0RBIT Lite

Description:
Fast responses.

Free.

------------------

0RBIT Core

Description:
Balanced performance.

Recommended.

------------------

0RBIT Pro

Description:
Advanced reasoning.

Premium.

Internally these models may all map to the same backend provider for now.

The user should never know.

# Chat Area

Implement:

- Streaming Responses
- Markdown Rendering
- Code Highlighting
- Tables
- Lists
- Blockquotes
- Inline Code
- Syntax Highlight

The conversation should feel polished.

Spacing should be excellent.

---

# Typing Animation

Do NOT use fake character-by-character typing.

Instead:

Display streamed tokens naturally.

Each response should progressively appear as it streams from the backend.

Maintain smooth scrolling.

---

# Chat Composer

Create a modern prompt composer.

Include:

Text Area

Send Button

Stop Generation

Attach File Button (UI placeholder)

Model Indicator

Suggested Prompt Button

Auto-growing textarea.

Maximum height before scrolling.

---

# Suggested Prompts

Before the first message,
display suggested prompts.

Examples:

Explain quantum computing simply

Write a Next.js API route

Summarize this article

Generate a marketing strategy

Create a Python script

Each suggestion becomes the user's first prompt.

---

# Response Actions

Each assistant response should support:

Copy

Regenerate

Like

Dislike

These actions should appear only on hover (desktop) or below the response (mobile).

---

# Markdown

Support:

Headers

Bold

Italic

Lists

Tables

Links

Task Lists

Code Blocks

Math support is optional.

---

# Code Blocks

Every code block should include:

Language Label

Copy Button

Syntax Highlighting

Rounded Container

Dark Theme

Long code should scroll horizontally.

---

# Auto Scroll

Automatically scroll while streaming.

If the user scrolls upward manually,

pause auto-scroll.

Resume only when the user returns to the bottom.

---

# Empty State

When there are no conversations,

display a premium welcome screen.

Include:

0RBIT Logo

Welcome message

Suggested prompts

Model selector

Keep it clean.

---

# Loading States

Use skeleton loading where appropriate.

Avoid blank screens.

Buttons should have loading indicators.

---

# Error Handling
