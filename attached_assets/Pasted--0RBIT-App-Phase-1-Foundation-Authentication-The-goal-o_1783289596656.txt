# 0RBIT App
## Phase 1 — Foundation & Authentication

The goal of this phase is to build the foundation of the 0RBIT application.

This is NOT the marketing website.

This project must be built as a completely separate application hosted on:

app.0rbit.ai

Do NOT reuse the marketing website layout.

The application should feel like a professional AI SaaS platform similar to ChatGPT, Claude, Cursor, or Vercel Dashboard, while maintaining the existing 0RBIT visual identity.

---

# Objective

Build the application shell, authentication flow, routing, navigation, and user foundation.

Do NOT implement AI chat yet.

Do NOT build API features.

Do NOT build Worker features.

Do NOT implement token economy.

Focus only on creating a solid production-ready application foundation.

---

# Tech Stack

Use the existing stack of this project.

Maintain clean architecture.

Write reusable components.

Organize the project for future scalability.

---

# Authentication

Use Privy as the authentication provider.

Supported login methods:

- Phantom Wallet
- Backpack Wallet
- Solflare Wallet
- Wallet Standard
- X (Twitter)
- Google

The login experience should be modern, fast, and frictionless.

No email/password authentication.

# Authentication Flow

User opens:

app.0rbit.ai

↓

If not authenticated

↓

Show Login Screen

↓

User selects Wallet or Social Login

↓

Authentication Success

↓

Redirect to Dashboard

If already authenticated:

Skip login completely.

Go directly to Dashboard.
